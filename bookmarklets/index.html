Bookmarklets

  • ANDI: ANDI is a "favelet" or "bookmarklet" that will:
    • Provide automated detection of accessibility issues
    • Reveal what a screen reader should say for interactive elements (the accessible name computation)
    • Give practical suggestions to improve accessibility and check 508 compliance
  • 44x44 Cursor: Changes your cursor to 44x44 pixels square for testing WCAG Target Size and Pointer Target Spacing requirements.
  • Text Spacing: Quick and dirty testing of WCAG 2.1 Success Criterion 1.4.12 Text Spacing.
  • Check Serialized DOM of Current Page: Sends a serialized version of the current state of the current DOM to the NuHTML checker.
  • Check for WCAG 2.X Parsing Conformance: Removes results from the NuHTML checker output that don't violate WCAG Success Criterion 4.1.1 Parsing.
  • Trigger character key shortcuts: Just automatically presses all the character keys to see if they trigger any changes on the page.
  • 3Hz Flasher: Just flashes some text on the screen at 3Hz, so you can have something to compare with to see if WCAG Success Criterion 2.3.1 Three Flashes or Below Threshold is being violated.
  • Label in Name Favlet: If it is different from the label, value, or textContent, this displays the accessible name for elements matching the following selectors: button, input, textarea, select, [tabindex='0'], a[href], summary. Helps in determining conformance with 2.5.3 Label in Name
  • HTML_CodeSniffer : A client-side script that checks HTML source code and detects violations of a defined coding standard.
  • Suggest autocomplete values : Adds a select element with all the possible autocomplete values whenever it sees an element that supports the autocomplete attribute. If an autocomplete attribute exists and it's value is an option in the select element, it will be marked as selected.
  • Isolate images:
    • Slightly experimental idea - this one does the opposite of some other tools (that hide images), and instead hides anything that is not an image
    • It zooms the page out so you can see more in one go - a wall of images
    • Aim of this is to be able to glance at the images and see if you can spot images that are actually just text which may not be obvious when scrolling the page in its normal state
    • Trying to weed out SC 1.4.5 Images of Text issues
  • Agnostic-axe : Developer tool that continuously observes the DOM to detect accessibility issues. Its audits are powered by axe-core.
